## Implementation Notes

### Key Decisions

1. **Checksum function**: Used `sha256_file()` from `src.utils.checksums` — CORRECT
   - NOT `snapshot_checksum()` (which hashes DataFrame bytes)
   - Consistent with Story 2-2 (Task 3) implementation

2. **DB query**: Used `db.list_snapshots(archived=False)` from Task 2
   - Returns list of dicts with all required fields
   - Filters out archived snapshots automatically

3. **Connection pattern**: Followed established pattern from `generate_ci_snapshot.py`
   ```python
   conn = db.connect(db_path="dados/data.db")
   try:
       snapshots = db.list_snapshots(archived=False, conn=conn)
   finally:
       conn.close()
   ```

4. **Output format**: Structured text with clear PASS/FAIL markers
   - Per-snapshot results
   - Summary section with counts
   - Explicit exit code in output

5. **Edge cases handled**:
   - No snapshots → exit 0 (nothing to validate)
   - Missing file → FAIL, count in summary
   - Checksum mismatch → FAIL, show expected vs computed
   - Incomplete metadata (no path or checksum) → skip silently

### Script Structure

Follows `scripts/validate_snapshots.py` pattern:
- Shebang: `#!/usr/bin/env python3`
- Module docstring with usage
- `sys.path` manipulation for imports
- `main()` function with clear return type
- `if __name__ == "__main__": raise SystemExit(main())`
